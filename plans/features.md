# Barkland Features Overview

Barkland is a multi-agent simulation dashboard that creates high-scalability isolated container sandboxes for simulated dog characters with unique personalities and needs.

---

## 🖥️ Dashboard & Simulation Control

The dashboard provides a modern glassmorphic interface that allows live-viewing of sub-agents and states.

*   **Live WebSocket Sync**: Renders tick counts, active dog states, and dashboard components triggers absolute live updating sequences.
*   **Start & Pause Controls**: Directly triggers layout simulation startup nodes inside backend threads to allocate dynamically.
*   **Autonomous Timer Control**: Support selector modes (10s, 30s, 60s, continuous) which hold a dashboard clock timer trigger backends using countdown countdowns executing automatic cooldown cleaning locks.

---

## 🐕 Dog Profiles & Agent Intelligence

Every dog has dynamic state trackers keeping its daily metrics and layout models fluid.

*   **Personality Flashers**: Dog boxes flash special icons revealing traits such as `JOCK`, `DRAMA_QUEEN`, or `PHILOSOPHER` upon startup frames.
*   **Circular Needs Trackers**: High-quality visual percentage pie charts rendering critical metrics:
    *   🍗 **Hunger**
    *   ⚡ **Energy**
    *   🎾 **Boredom**
*   **State Status Cycle Tracker**: Distinct state borders change colors describing standard action slots like `SLEEPING` (purple), `EATING` (green), and `PLAYING` (pink).
*   **Generative Bark Box**: Background agents formulate unique funny lines inside floating speech bubbles linked directly to profiles.

---

## ☁️ Sandbox claim Controller Stream

A terminal tracker streams directly on lower widgets allowing visual lookup node triggers executing backend setup binds.

*   **Claim Lifecycle List**: Dynamic live-feed tracker streaming allocation bindings.
*   **Metadata Lookup**: Layout tracking assigned Sandbox claim IDs and IP endpoints (e.g., `sandbox-claim-*` with `10.x.x.x`).
*   **Status Indicators Counter**: Dynamic tracker counters summing state bounds aggregate statuses: `Running`, `Created`, `Paused`.
